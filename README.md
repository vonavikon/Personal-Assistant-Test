# Executive Assistant — Персональный AI-ассистент руководителя

Шаблон репозитория для создания персонального AI-ассистента на базе **Claude Code + VS Code**. Позволяет организовать рабочее пространство руководителя: управление контактами и встречами 1-1, проектами, решениями, протоколами, а также вести базу знаний с семантическим поиском.

## Возможности

- **Контакты и люди** — карточки сотрудников, стейкхолдеров, внешних контактов
- **Встречи 1-1** — подготовка, проведение, фиксация договоренностей
- **Проекты** — отслеживание активных проектов, бэклог, архив
- **Решения** — журнал решений (ADR), политики, обоснования
- **Протоколы** — обработка стенограмм встреч, извлечение задач и решений
- **База знаний** — методологии, процессы, глоссарий с семантическим поиском (qmd)

## Требования

- **VS Code** с расширением Claude Code
- **Claude Code** CLI (устанавливается вместе с расширением)
- **Node.js 18+** — для qmd (опционально, для семантического поиска)

## Быстрый старт

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/mdemyanov/executive-assistant.git my-vault
   cd my-vault
   ```

2. **Откройте в VS Code:**
   ```bash
   code .
   ```

3. **Запустите инициализацию:**
   В чате Claude Code выполните команду `/init` — ассистент создаст структуру папок и настроит рабочее окружение.

## Доступные команды

После инициализации доступны следующие команды в чате Claude Code:

| Команда | Описание |
|---------|----------|
| `/find-person` | Поиск информации о человеке в базе знаний |
| `/new-1-1` | Создание протокола встречи 1-1 |
| `/new-decision` | Фиксация нового решения (ADR) |
| `/new-project` | Создание карточки проекта |
| `/kb-stats` | Статистика по базе знаний |
| `/meeting-debrief` | Обработка стенограммы встречи |

## Структура хранилища

Используется система **Johnny Decimal** для организации файлов:

```
00_CORE/          — идентичность, стейкхолдеры, стратегия
10_PEOPLE/        — карточки людей и контактов
20_MEETINGS/      — протоколы встреч, комитеты, стендапы
30_PROJECTS/      — проекты (активные, бэклог, архив)
40_DECISIONS/     — решения, политики, журнал
50_KNOWLEDGE/     — методологии, процессы, глоссарий
60_DOMAIN/        — доменная область (настраивается)
90_TEMPLATES/     — шаблоны документов
99_ARCHIVE/       — архив стенограмм и завершенных проектов
```

## Инструменты

### qmd (опционально)

Семантический поиск по базе знаний. Позволяет находить релевантные документы по смыслу, а не только по ключевым словам.

```bash
# Установка
bash scripts/install-tools.sh --qmd
```

### tg-parser (опционально)

Парсер Telegram-чатов для импорта переписки в базу знаний.

```bash
# Установка
bash scripts/install-tools.sh --tg-parser
```

### Установка всех инструментов

```bash
bash scripts/install-tools.sh --all
```
