# Поиск информации о человеке

## Алгоритм

### 1. Определи тип ввода

| Ввод | Действие |
|------|----------|
| ID (латиница: vshadrin) | Прямой поиск файла |
| Имя (кириллица/латиница) | Семантический поиск → извлечь ID |

### 2. Поиск по ID

```
Read: 10_PEOPLE/{id}/{id}.md
```

### 3. Поиск по имени (fallback)

Если qmd доступен:
```
qmd_deep_search: query="{name}", collection="{vault}-people", n=5
```

Если qmd недоступен:
```
Grep: pattern="{name}", path="10_PEOPLE/"
```

Из результата извлеки ID из пути: `10_PEOPLE/{id}/` → `{id}`

### 4. Собери связанную информацию

```
# 1-1 встречи
Glob: 10_PEOPLE/{id}/1-1/*.md

# Проекты
Grep: pattern="{id}", path="30_PROJECTS/active/"

# Упоминания
Grep: pattern="{id}", path="."
```

## Формат ответа

```markdown
## {Полное имя}

**ID:** {id}
**Должность:** {role}
**Подчинение:** {reporting}

### Профиль
[[10_PEOPLE/{id}/{id}]]

### Текущие проекты
- {список}

### Недавние 1-1
- {список последних 3-5}

### Контекст
{заметки из профиля}
```

## Золотое правило

**ID > Name**: Поиск по ID работает 100%, по именам — через qmd_deep_search или Grep.
