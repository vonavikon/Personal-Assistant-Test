# /process-transcript - Обработка транскриптов ktalk

> **Анализ и структурирование встреч из Naomi Talk**

---

## Что делает эта команда

1. Получает транскрипт встречи из ktalk MCP
2. Анализирует через LLM
3. Извлекает: участники, темы, решения, задачи
4. Заполняет шаблон встречи
5. Создаёт профили новых участников
6. Проводит интерактивное закрепление

---

## Использование

```
/process-transcript
```

Или укажите конкретную запись:
```
/process-transcript --id <recording_id>
```

---

## Схема ID

```
ID = {первая_буква_имени}{фамилия_транслит}
```

Примеры:
- Демьянов Максим → `mdemyanov`
- Иванов Константин → `kivanov`
- Зыков Петр → `pzykov`

---

## Алгоритм работы

### Шаг 1: Получение данных

**Если указан recording_id:** Получите через ktalk MCP

**Если не указан:** Покажите список последних записей

### Шаг 2: Анализ транскрипта

Извлеките сущности:

```yaml
extracted:
  title: "Краткое название"
  date: YYYY-MM-DD
  duration: "1h 30m"
  meeting_type: sprint-planning|masterclass|sync|1-1|committee|other

  participants:
    - name: "Фамилия Имя"
      role: "Роль"
      company: "Компания"

  topics: []
  decisions:
    - what: "Что решено"
      who: "ID или имя"
      when: "Срок"
  actions:
    - task: "Задача"
      owner: "ID или имя"
      deadline: "YYYY-MM-DD"
  summary: "2-3 предложения"
```

### Шаг 3: Создание файла встречи

Путь: `20_MEETINGS/ktalk/YYYY-MM-DD_{title}.md`

```markdown
---
type: meeting
id: meeting-{timestamp}
title: "{Название}"
date: {YYYY-MM-DD}
meeting_type: {тип}
tags: [ktalk]
participants: [{id1}, {id2}]
---

# {Название}

**Дата:** {дата}
**Тип:** {тип}
**Участники:** [[10_PEOPLE/{id1}/{id1}|{name1}]], [[10_PEOPLE/{id2}/{id2}|{name2}]]

## Краткое содержание
{summary}

## Темы
1. {topic1}
2. {topic2}

## Решения
| Что | Кто | Когда |
|-----|-----|-------|
| {decision} | [[10_PEOPLE/{id}/{id}|{name}]] | {when} |

## Задачи
| Задача | Ответственный | Срок |
|--------|---------------|------|
| {task} | [[10_PEOPLE/{id}/{id}|{name}]] | {deadline} |

## Минутки
<!-- Заполняется через интерактивное закрепление -->
```

### Шаг 4: Создание профилей участников

Для каждого нового участника:

1. Проверьте существование:
```bash
ls 10_PEOPLE/{id}/{id}.md
```

2. Если нет — создайте через `/new-project` или вручную

### Шаг 5: Интерактивное закрепление

```
У вас есть минутки с этой встречи?
```

Если нет — мини-ретроспектива:
```
1. Что главное произошло?
2. Какие решения зафиксировали?
3. Что осталось непонятным?
4. Есть ли блокеры?
```

```
Пользователь: /process-transcript

Какую обработаем?

Пользователь: 1

Найдены участники:
- mdemyanov (существует)

Решений: 2
Задач: 5

```
